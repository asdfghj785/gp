# 🎯 离岸量化工作站 1.0：架构与开发者指南

## 1. 项目概述 (Project Overview)
本项目是一个部署在 macOS 环境下的本地化、全自动量化交易辅助系统。其核心架构基于**“正交因子分离”**理念，实现了从底层数据、算法选股、情绪风控到自动化运维的 24 小时交易闭环：
* **技术/资金面（主观动量）**：通过 XGBoost 模型对东方财富全市场高频数据进行日内胜率预测，并支持盘中一键自动筛选推送。
* **基本/情绪面（微观舆情）**：通过抓取个股公告与散户社区（股吧），调用本地大模型进行实时情绪风控，彻底消灭“黑盒交易”。
* **无人值守自动化**：包含心跳监测、实盘狙击推送、盘后全量数据清洗落库等守护进程。

## 2. 核心技术栈 (Tech Stack)
* **主工作目录**：`/Users/eudis/ths`
* **数据源**：东方财富 Web API (实时行情 `push2`，股吧微观情报)。
* **机器学习核心**：XGBoost (`overnight_xgboost.json`)。
* **本地大模型 (LLM)**：Ollama 驱动的 `qwen2.5:14b`。
* **后端 API**：Python + FastAPI + Uvicorn。
* **前端 Web**：Vue3 + Vite + TailwindCSS。
* **历史数据库**：按股票代码分离的 Parquet 文件型列式数据库。
* **消息推送**：PushPlus (微信心跳与选股推送)。
* **自动化调度**：macOS `LaunchAgents`。

## 3. 全景目录结构与模块详解 (Architecture)

系统分为四大独立运作又相互配合的“子系统”：

### A. 核心驱动层 (Model & Core)
系统的“大脑”，负责算法逻辑和基准规则。
* `overnight_xgboost.json`：**核心算法权重**。经过大量历史 K 线训练出的 XGBoost 模型文件，负责在实盘中计算日内胜率。
* `save_model.py`：**模型训练工厂**。用于重新训练历史数据并覆盖更新 XGBoost 模型文件，实现策略迭代。

### B. 自动化巡航层 (Automated Monitoring)
系统的“无人值守”护城河，负责盘中自动化和系统维稳。
* `heartbeat_monitor.py`：**系统心跳监测仪**。挂载在后台定时运行，通过 PushPlus 微信推送发送“系统存活”信号，确保离岸系统未宕机。
* `daily_sniper.py`：**日内头狼狙击手**。在盘中（如尾盘 14:40）自动运行，调用 XGBoost 扫描全市场，筛选出当日**胜率最高的一只股票**，通过 PushPlus 推送到微信，作为次日早盘博弈标的。

### C. 盘后数据基建层 (Data Infrastructure)
系统的“血液循环”，负责数据的收集、清洗和归档。
* `data/all_kline/`：**列式历史数据仓**。核心数据存储区，包含 5000+ 个以 `股票代码_daily.parquet` 命名的文件，读取极快，完美适配量化回测。
* `data_recorder.py`：**盘后全量同步引擎**。每日 16:05 由 LaunchAgents 自动触发。具备反爬虫分页伪装（防 502 报错）与节假日智能休眠机制。抓取当日全市场 K 线并以追加 (Append) 形式单行写入 Parquet 仓库。
* `check_data.py`：**数据探针**。运维诊断工具。用于手动抽查指定股票的底层 Parquet 文件，验证每日最新数据是否成功缝合落库。
* `merge_history.py`：*(备用/已退役)* **数据融合手术刀**。用于将外部两年的历史老数据，清洗、对齐并导入现有的底层数据库。

### D. 可视化交易终端层 (Web Dashboard)
最新开发的“白盒透视”平台，用于人工介入和逻辑验算。
* `stock_xray.py`：*(终端版)* **纯命令行情绪 X 光机**。支持在终端一键传入股票代码，调用 Ollama 输出舆情文字报告。
* `quant_dashboard/`：**Web 可视化综合面板**。
    * `backend/main.py`：FastAPI 后端服务（端口 8000）。桥接实盘数据流、调用本地大模型，提供雷达与舆情 API。
    * `frontend/src/App.vue`：Vue3 前端 UI（端口 5173）。提供实盘 Top 50 股票雷达榜单，并通过侧边抽屉展示 XGBoost 特征贡献度与大模型抓取的情绪证据链。

## 4. 核心工作流与多脚本协同生态 (Workflows)

系统 1.0 已形成 24 小时闭环运作的黑客级交易生态：

* **阶段一：全天候底座防御（24H）**
  `heartbeat_monitor.py` 定时向微信发送心跳包，提供系统存活证明，防止断电断网导致的漏单盲区。

* **阶段二：日内白盒选股与情绪裁决（盘中 09:30 - 15:00）**
  用户通过 Web 看板实时监控。后端调用模型算出胜率 Top 50 并前端渲染；用户点击“深度透视”唤醒 Qwen 模型爬取东财股吧（公告、资讯、散户发帖），AI 强制输出“绿灯/红灯”指令及 3 条原始证据链，辅助人工下单。

* **阶段三：尾盘自动化一击致命（盘中 14:40 - 14:55）**
  定时任务触发 `daily_sniper.py`，后台静默运行扫盘逻辑，锁定全市场胜率最高标的，通过 PushPlus 直推微信，实现离线接收绝杀指令。

* **阶段四：盘后数据沉淀与清算（盘后 16:05）**
  LaunchAgents 触发 `data_recorder.py`。智能识别交易日，分页抓取大盘数据，以 `date` 字段严格去重追加到 Parquet 文件池，并推送入库报告，实现数据反哺模型的复利循环。
